# Next.js Cross-origin HMR Blocked

**Trigger**: Running Next.js dev server and accessing it from another device on the local network (e.g., using `192.168.x.x` from a mobile phone or secondary tablet).
**Symptom**: Next.js logs `Blocked cross-origin request to Next.js dev resource /_next/webpack-hmr from "192.168.x.x".` The UI crashes or HMR fails to load because cross-origin access to Next.js dev resources is blocked by default for safety.
**Root Cause**: Next.js 15+ enforces strict origin checks for development assets to prevent local network attacks.
**Fix Pattern**: 
Add the specific IP or origin to `allowedDevOrigins` in the Next.js config.
```typescript
// next.config.ts
const nextConfig: NextConfig = {
  // @ts-ignore if types are outdated
  allowedDevOrigins: ["192.168.x.x"],
};
```
